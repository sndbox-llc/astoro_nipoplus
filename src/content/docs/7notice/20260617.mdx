## iOSをお使いの方は起動時に通常より異常に時間がかかることがあります

Firebase側のバグにより、iOSをお使いの方は起動時に通常より異常に時間がかかることがあります。

以下の条件下では、ロードに時間がかかりすぎることが報告されています。

- 「MacOSまたはIOSであること」
- 「Safari Version26.4以降のバージョンを使用していること（NipoPlusは内部でSaafriを使用しているため該当してしまいます）」

必ず発生するわけではなく、発生するときと発生しないときがあるとのことです。
一度ロード遅延が発動すると、１度のリクエストに３０秒ほど（連続で取得が走る場合は更に上乗せされる）のフリーズに近い状況が起きるとのことです。

現在、本家Firebaseのバグ修正を待っている状況ですが、現時点でまだ修正や進捗の投稿が無い状況です。

実際に報告が上がっている議題ページです（英語）

https://github.com/firebase/firebase-js-sdk/issues/9789

## 影響範囲

Nipo / NipoPlusをお使いのiOSユーザーの方が影響を受ける可能性があります。

## 一時的な回避策の提案

発生する条件が iOS / Macで、なおかつSafariを使用することであるため、Google Chromeなどの異なるブラウザを使うことで、この問題を回避できます。
iPhoneやiPadをお使いのお客様は、App StoreのNipoPlusを使用すると、内部的にSafariが組み込まれているため、起動遅延の問題が発生してしまいます。
そのため、iPhoneやipadにGoogle Chomeなどのブラウザをインストールいただき、そちらから起動していたくことでこの問題は回避可能です。
ご不便をおかけして申し訳ありませんが、修正までの間はこの回避策をご検討いただければと思います。

## NipoPlus 一時的な応急処置を実施

2026年6月24日  
NipoPlus Version 1.84.0にて、iOSで起動時に時間がかかるバグの応急処置を実施しました。これは応急処置であり、根本的な解決ではありませんが、起動時のロード時間を短縮することが可能です。
根本的解決までは、引き続きFirebase側の修正を待つ必要があります。
進捗があり次第本ページにてお知らせいたします。

ご不便をおかけして申し訳ございませんが、修正までの間はこの応急処置をご利用いただければと思います。
