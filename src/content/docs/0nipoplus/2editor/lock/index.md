提出先の固定をすることで、スタッフは日報作成時に提出先を指定する必要がなくなります。  
スタッフの操作が簡略化され、現場の混乱も軽減できるため設定を**強く推奨**します。  
この設定で頻出する「提出先」や「共有先」について詳しくは[提出先と共有先の概念](/nipoplus/gainen/destination/)をご覧下さい。

## 提出先の固定方法は2種類 {#submission_priority}

「スタッフ単位で指定」するか、「テンプレート単位で指定」する方法があります。条件が重複している場合は**スタッフ単位が優先**されます。

<dl class="basic">
<dt><a href="#fixed_staff">スタッフ単位で指定</a></dt>
<dd>最も優先。特定のスタッフの提出先を細かく制御したい場合に利用します。</dd>
<dt><a href="#fixed_template">テンプレート単位で指定</a></dt>
<dd>2番目に優先される設定です。特定のテンプレートで書かれた日報の提出先を固定したい場合に利用します。</dd>
</dl>

:::note[設定がない場合]
設定がない場合は、自動で前回の提出先がセットされます。過去の提出状況もない場合は空欄になります
:::

## テンプレート単位で提出先を固定（推奨） {#fixed_template}

実務に沿っているためこちらの方法をおすすめします。例えば「不具合報告日報はAさんに出す」といった運用に最適です。

1.  左メニューから「グループ設定」をクリック
2.  上部メニューの「テンプレート管理」をクリック
3.  提出先を指定したいテンプレートの行にある「提出先の指定」ボタンをクリック
4.  表示される画面で、提出先や共有先を指定し、設定を保存

![テンプレート単位でスタッフの提出先を固定する](./img/template-dist-lock_tablet.webp)

## スタッフごとに個別に提出先を指定する {#fixed_staff}

スタッフが特定のテンプレートを使った場合に特定の相手に提出させるという使い方です。
計算上はスタッフ数×テンプレート数の設定が可能ですが、全て設定する必要は有りません。

:::tip[例えば？]
Aさんが「業務日報」」というテンプレートを使った場合は「X部長宛」に固定するといった使い方ができます
:::

設定手順：

1.  左メニューから「グループ設定」をクリック
2.  上部メニューの「スタッフ管理」をクリック
3.  提出先を指定するスタッフの行にある「提出先の固定」ボタンをクリック
4.  ポップアップで設定画面が表示されます

![テンプレート単位の設定があったとしてもこっちが優先される](./img/dist-setting2_tablet.webp)

![【誰が】【どのテンプレートを使ったとき】【誰に出すのか】を設定する](./img/dist-detail2_tablet.webp)

必要に応じて、以下のオプション設定も可能です。

<dl class="basic">
<dt>提出先や共有先の変更を許可しない</dt>
<dd>ONにすると日報作成時に、スタッフが提出先や共有先を変更できなくなります</dd>
<dt>グループ内の全スタッフを自動で含める</dt>
<dd>共有先にグループ内の全スタッフが自動でセットされます（つまり、全員が閲覧可能になります）</dd>
<dt>リセット</dt>
<dd>この設定で指定した提出先、共有先の情報をすべてリセットします。</dd>
</dl>
